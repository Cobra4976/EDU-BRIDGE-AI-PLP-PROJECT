### 4. Firebase Setup

1. Create project at [console.firebase.google.com](https://console.firebase.google.com)
2. Enable **Authentication** â†’ Email/Password
3. Create **Firestore Database** with collections:
   - `users` - User accounts (students/teachers)
   - `students` - Student profiles
   - `teachers` - Teacher profiles
   - `schools` - School licenses
   - `subscriptions` - Premium subscriptions
   - `transactions` - Payment records
   - `posts` - Community posts
   - `replies` - Post replies
   - `userCommunityStats` - User engagement stats

4. Add **Firestore Security Rules**:
```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Users can read/write their own data
    match /users/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
    
    match /students/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
    
    match /teachers/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
    
    match /subscriptions/{userId} {
      allow read: if request.auth != null && request.auth.uid == userId;
      allow write: if request.auth != null;
    }
    
    // Community posts - public read, authenticated write
    match /posts/{postId} {
      allow read: if request.auth != null;
      allow create: if request.auth != null;
      allow update, delete: if request.auth != null && resource.data.authorId == request.auth.uid;
    }
    
    match /replies/{replyId} {
      allow read: if request.auth != null;
      allow create: if request.auth != null;
      allow update, delete: if request.auth != null && resource.data.authorId == request.auth.uid;
    }
    
    match /userCommunityStats/{userId} {
      allow read: if request.auth != null;
      allow write: if request.auth != null;
    }
    
    match /transactions/{transactionId} {
      allow read: if request.auth != null && resource.data.userId == request.auth.uid;
      allow create: if request.auth != null;
      allow update: if request.auth != null;
    }
    
    match /schools/{schoolId} {
      allow read: if request.auth != null;
      allow write: if request.auth != null && 
        (resource.data.adminId == request.auth.uid || !exists(/databases/$(database)/documents/schools/$(schoolId)));
    }
  }
}
```

## ğŸ› Troubleshooting

### Firebase Connection Issues
**Problem**: "Failed to load resource: 400"
**Solution**: 
- Check Firebase config in `.env`
- Verify Firestore security rules allow read/write
- Clear browser cache and cookies

### Gemini API Rate Limit (429)
**Problem**: "API request failed: 429"
**Solution**:
- Wait 1-2 minutes before retry
- Upgrade to Gemini paid tier for higher limits
- Implement exponential backoff (already added)

### M-Pesa Payment Not Working
**Problem**: STK push not received
**Solution**:
- Verify M-Pesa credentials in backend `.env`
- Check callback URLs are publicly accessible
- Use ngrok for local testing
- Test with Safaricom sandbox phone: 254708374149

### Offline Mode Not Working
**Problem**: Content not accessible offline
**Solution**:
- Check service worker is registered (browser DevTools â†’ Application)
- Verify IndexedDB has cached data
- Clear site data and re-cache content
- Use HTTPS (required for service workers)

### Subscription Not Updating
**Problem**: Premium features not unlocking
**Solution**:
- Check `subscriptions` collection in Firestore
- Verify M-Pesa callback was received (check backend logs)
- Manually trigger subscription check in dashboard
- Check browser console for errors
